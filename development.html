Morphed Audio

At one point of time on our research we realized that voice cloning will be really difficult to implement as the technology which can clone a realistic sounding output voice with only 2 hours of input audio is not available yet. So we looked into a non Dynamic audio solution The idea is to record the original audio message, ages number between from 1-100, and weather types like sunny, rainy fog etc and record then 200-300 names which will cover 60-70% of PSV's data base and then morph them together to send a personalized message.

We personalize the message with 4 factors:
  1. Name: Name of the Fan
  2. Age: current age of Fan
  3. City: The city he lives in according to PSV's Database
  4. Weather: Current weather in the Fan's city

It looks really promising as we have all his information and we can easily calculate his age and find out the Weather in the city. For getting the weather I used openWeather API and which give me lot of weather details for the given city that can be used in the output Audio. Here is an example output audio of the prototype. It was built by myself and Niek where Niek focused on the Audio part and I mainly focused on weather aspect.

YouTube API Upload

As email does not support video attachment it was a challenge for me to find some hosting platform where I can host the videos either free of cost or at minimal cost. I talk more about this in the Research section. YouTube was the first option that came to my mind and fortunately there is an API available to upload the video.
As the whole process from combining video to sending the email is automated, video upload needs to be done automatically. And for that I created a prototype that shows that it is possible to upload videos to YouTube using Google developers API.
The process was simple enough I had to register myself as a developer in Google console website and create an application which gave me my API credentials i.e. Client ID and Client Secret. On your right is a sample screenshot of my YouTube channel, This video is uploaded via API using the prototype I built, And as you can see it is scheduled for a given date and time which is one of our major requirements.

Data flow Diagram

Check SAD document for better results

This diagram show how data is handled by different applications. This is how it works:

  1. Day 1: Before Birthday
    • Prototype: Fetch fans from PSV database who have birthday next day
    • Prototype: It generates audio file i.e. fan's name and age using Google TTS API it translates text into audio
    • Prototype: Fetches audio file generated usingTTS and embed it in Pre recorded video and render the video
    • Vimeo: fetched rendered video, upload it to Vimeo account and return the upload link where video is stored
    • Prototype: Combine video link (a landing page for the video) in the birthday email using group ID from PSV database
  2. Day 2: On Birthday
    • PSV Mailing service: Send emails to people who have their birthday
    • Fan: Fan receives an email with the personalized video.
    • Fan: opens the link
    • Prototype: Search the video with the given group ID
    • PSV server: acts as a landing page show video to fan.

All the C-4 models have been made during the internship and can be found in the Software Architecture Document.

As I and Mathijs (software developers) were familiar with Java, we have implemented major part of this prototype in Java. One of my goals during the internship was to code in Python so I did some Python programming as well and some part is built in python.

Our Prototype does 4 major parts:

  1. PSV Database
  2. TTS
  3. Video Render from AfterEffects
  4. Hosting

PSV Database

In our Prototype we are successful in calling PSV's Database and make call to get Fan data. Now we can successfully fetch all the people who have their birthday's on the present day. I worked on this separately to access the database and see what details/data is available to us that can be useful on the project.

TTS

Using our, API we can successfully call Google TTS API to generate Audio files, that are personalized for each fan and generated in order to embed them in the raw video of a player to deliver a personalized video in the email.

Video Render

This is the part where video editing happens, Editing can be automated in afterEffects using ExtendScript, The language is JSX(Java script extended), THis part is done by the media designer in the team, Niek. He created a script that gets input from our prototype and combine audio samples with the raw video and finally render a personalized video. This sample prototype is working and we are able to render video automatically, but this part was not my specialty as I was looking more into other 3 main parts i.e. TTS, Hosting and PSV Database.

Hosting

For hosting we are using Vimeo Business Plan which gives us 5 TB of Monthly hosting space on cloud and also their is an API available to upload the videos. In the prototype I built a functionality that uploads the videos to the business account, using Vimeo's API. It is possible to fetch the rendered video from location then upload it to Vimeo and finally receive the link where the video is hosted.
I was completely responsible for this part and everything was done by me individually.

Challenges

Building prototypes was a fun process, but I wish we had much time for testing purposes, as we are already done with our internship we did not get any chance to test our product, although we did tested what we have with sample users to make sure we are on the right track. Here are some challenges that I faced during the project:

  • Client: We had very good communication with our client, but it was difficult to make them happy, As a result it took long time to come to a conclusion and we entered the development phase late in mid-December
  • Technology: We explored a lot of technologies, but to satisfy the requirement of voice cloning with recording time constraints, It ws just not possible to develop a voice clone and we had to move to a different concept.
  • Work from Home: Due to corona virus we had were forced to work from home, My home Office is not as good a sthe office so it was a bit challenging to do everything from home.
  • Costs: Cost was a big factor, we had to find the cepaest solution, and cheapest is not always the best. We had to discrd some really cool ideas because of cost.
